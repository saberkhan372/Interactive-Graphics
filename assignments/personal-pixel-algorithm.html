Assignment · Unit 0

Personal Pixel Algorithm

Before any code: design an image on graph paper, then write a numbered algorithm that another human could follow to recreate it exactly. This is the course's first project, and its quietest big idea — a picture can be transmitted as instructions, which means a picture can be data.

As assigned 2022–23 · bring to class Thursday, September 22 · typed and printed.

What to do

  • Create a design using graph paper, on the theme of computers;
  • Use any colors, but fill entire pixel boxes — no partials;
  • Write an algorithm for someone else to recreate your design;
  • Number your steps;
  • Type it up and print it out;
  • Bring it to class.

Consider, as the brief put it

  • What important information do we need to consider?
  • How should we note details about a pixel?
  • How do we identify each pixel?
  • What sort of algorithms will be easiest for humans to follow?

Why it matters

Everything later in the course — BASIC plotting, p5.js loops, the graphics pipeline — is a machine answering these same four questions faster. Students who struggle to describe "the third pixel in the second row" in September recognize PLOT 2, 1 in October.
