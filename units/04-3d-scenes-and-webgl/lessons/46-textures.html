Unit 4 · Lesson 4.6

Textures

A texture lets an image become part of a 3D surface. The lesson asks students to notice that a surface can carry color from a material, light from the scene, or image data loaded from somewhere else.

Source deck: IG 3.6 - Textures.pptx · 5 slides

What the class did

The warm-up returned to colors and materials in p5.js WebGL. The lesson then introduced texture as an image wrapped around a shape. The basic pattern was to load the image in preload, replace a material command with texture, and test the result on different shapes.

The deck also pointed toward camera capture as a possible texture source. That matters because a texture does not have to be a static file; it can be a changing image source mapped onto geometry.

Code shown

let img;

function preload() {
  img = loadImage('texture.png');
}

function draw() {
  texture(img);
  box(140);
}

Key idea

A texture is image data assigned to a surface. It changes what the object shows without changing the underlying geometry.
