Unit 4 · Lesson 4.1

GitHub Pages

Publishing changes the audience for student work. A project that lives only in an editor is private by default; a GitHub Pages site asks students to decide how they will present their process, code, and visual results to someone else.

Source deck: IG 3.1 - Github pages.pptx · 9 slides

What the class did

Students discussed the steps for setting up a GitHub Pages site and how to present work on that site. The portfolio needed a landing page, project pages, explanations, links, samples of work, images or video, and navigation back to the landing page.

The lesson also asked students to use JavaScript, HTML, and CSS to make the site individualized and presentable. The site had to hold the first-semester projects before the WebGL scene arrived.

Key idea

Publishing is part of the technical work. A clear website requires structure, navigation, media choices, and enough writing for the project to be understood outside the classroom.
