Unit 4 · Lesson 4.7

Camera and Perspective

The camera is not a passive window. It is a system of choices about where the viewer is, what the viewer looks at, and how depth is converted into a flat image.

Source deck: IG 3.7 - Camera & Perspective.pptx · 5 slides

What the class did

The warm-up asked how textures are applied and what kinds of things can be used as texture sources. The lesson then moved into camera and perspective commands: perspective, ortho, frustum, pan, tilt, lookAt, and camera with eye position, center position, and up vector.

This is the point where the viewer becomes part of the scene. Changing the camera can alter the meaning of the same objects without moving the objects themselves.

Code shown

perspective(PI / 3, width / height, 0.1, 5000);
camera(
  0, -160, 500,
  0, 0, 0,
  0, 1, 0
);

Key idea

The same scene can be rendered differently because the camera changes the relationship between world space and the image plane.
