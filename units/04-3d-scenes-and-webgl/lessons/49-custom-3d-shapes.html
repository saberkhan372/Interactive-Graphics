Unit 4 · Lesson 4.9

Custom 3D Shapes

A custom shape removes the comfort of built-in primitives. Students have to decide which vertices exist, in what order they should be connected, and how those decisions produce a surface.

Source deck: IG 3.9 - Custom 3D shapes.pptx · 4 slides

What the class did

The warm-up asked how camera options affect what is seen on screen. The lesson then moved toward custom geometry, using the same projection vocabulary from the previous lesson while preparing students to build shapes through explicit vertices instead of only through box or sphere.

Even when a slide deck is short, the curricular purpose is strong: the final WebGL project requires students to make at least one custom shape, so they need to understand that a 3D object can be authored point by point.

Code shown

beginShape();
vertex(-80, -80, 0);
vertex( 80, -80, 0);
vertex( 0,  80, 60);
endShape(CLOSE);

Key idea

Custom geometry is ordered information. The computer connects vertices in the order the program gives them, so the list itself becomes the shape.

Try it

Open Vertex Sculptor and place three vertices. Then add a fourth and watch how the shape changes because the order changed the surface.
