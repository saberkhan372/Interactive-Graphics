Unit 3 · Lesson 3.1

The Graphics Pipeline

The graphics pipeline is a sequence of changes in representation. A model begins as vertices in its own coordinate system, moves through transformation and camera space, is projected to a flat plane, and finally appears as marks on a screen.

Source doc: 3d pipeline.docx

What the class did

The source document explains the pipeline through matrix multiplication, rotation matrices, and 3D projection. Students read objects as collections of vertices, transformations as matrix operations applied to those vertices, and projection as the step that maps 3D coordinates onto a 2D screen.

The wireframe rendering section then gives the full sequence: modeling, transformation, viewing transformation, projection, wireframe rasterization, and output to screen. The point is not to memorize vocabulary. The point is to understand that every rendered line has passed through a chain of numerical changes.

Code shown

function project(point, distance) {
  const scale = distance / (distance + point.z);
  return {
    x: point.x * scale,
    y: point.y * scale
  };
}

Key idea

The pipeline makes 3D visible by changing coordinates in stages. Each stage has a job, and confusing those jobs makes the code harder to debug.

Try it
