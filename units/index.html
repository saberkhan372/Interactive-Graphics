The Six Units

A six-part web reconstruction drawn from both years of the course, not a transcript of either cohort's exact calendar.

The original materials were organized differently across 2022–2023 and 2023–2024. This archive uses a composite structure so the underlying argument is easier to see: history and constraint first, then BASIC and recursion, then the move from flat images to 3D space, then the pipeline, WebGL scenes, and finally the game-development branch.

Unit 0 · History and Constraint

What hardware would and wouldn't let you make

Apple II, NES, Bryce3D, and today. A history that starts with the teacher's own life and ends with the student's own hands on the constraint.

→ Tools: Constraint Machine, Pixel Algorithm Playground
Unit 1 · BASIC and Fractals

Applesoft BASIC, bouncing balls, and recursion made visible

Programming on an emulator of a 1977 computer, then bridging the exact same idea into p5.js. A fractal tree shows what recursion actually does.

→ Tools: BASIC ↔ p5.js Twin Panels, Recursion Made Visible
Unit 2 · 2D to 3D

Adding a third dimension, on purpose

The same shape, flat then extruded, makes the jump from 2D to 3D a felt action, not an abstract leap.

→ Tool: Flatten/Extrude
Unit 3 · Pipeline and Wireframe

The math under WebGL

Building a 3D renderer without the library doing it for you, from model space to screen space.

→ Tool: Pipeline Step-Through
Unit 4 · 3D Scenes and WebGL

A full scene, built up piece by piece

Lighting, materials, camera and perspective, custom shapes built vertex by vertex, and textures: the pieces of a real 3D scene, isolated one at a time.

→ Tools: Light & Material Lab, Camera & Perspective Lab, Vertex Sculptor
Unit 5 · Game Dev Bridge

Pixel art skills meet GB Studio

The bridge from p5.js pixel work into a dedicated game engine, keeping the same 4-color-constraint thinking from Unit 0.

→ Tool: Game Boy Palette Sprite Editor
