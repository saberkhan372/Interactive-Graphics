Unit 2 · Lesson 2.0

Intro to WebGL in p5.js

The first WebGL lesson changes the renderer before it changes the student's thinking. A browser sketch still has HTML, CSS, and JavaScript, but the canvas now has depth, rotation, and a coordinate system that can no longer be read as only left-right and up-down.

Source deck: 2.0 - Intro to WebGL in p5.js.pptx · 9 slides

What the class did

Students reviewed the Unit 0 question of whether directions can be optimized, then returned to the structure of a browser sketch: index.html, sketch.js, p5.js, and the WebGL renderer. The lesson framed WebGL as the start of a longer 3D sequence: geometry, light and materials, texture, camera and perspective, offscreen graphics, models, custom shapes, and eventually a project.

The first coding move was small but decisive: the third argument to createCanvas changes the rendering context. From that point on, shape, position, and rotation have to be understood in a space with depth.

Code shown

function setup() {
  createCanvas(400, 400, WEBGL);
}

function draw() {
  background(240);
  rotateY(frameCount * 0.01);
  box(120);
}

Key idea

Adding WebGL does not remove the old web structure. It changes what the drawing surface can represent. The file system stays familiar while the space inside the canvas becomes less familiar.
