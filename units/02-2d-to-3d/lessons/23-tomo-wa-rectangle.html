Unit 2 · Bridge lesson

Tomo Wa 3D Rectangle

This bridge lesson makes translation difficult in the right way. Students have to read an Applesoft BASIC program that already contains trigonometry, projected points, and edge drawing, then rename and rebuild it in JavaScript so the structure becomes legible.

Source doc: 3D Rectangle from Tomo Wa on Applesoft BASIC in JavaScript.docx

What the class did

The source document begins with a clear to-do list: add comments to show understanding, update variable names to show what they do, and recreate the code in p5.js. The original program uses sine and cosine to predict points on a circle, turns those values into projected screen coordinates, draws the edges with HPLOT, then increments the angles and repeats.

This is the same translation habit from Unit 1, but now the translation has more conceptual pressure. Students are no longer only translating syntax. They are also explaining how a rotating wireframe can be built from projected points and connected edges.

Code shown

60 E = COS(A)
70 F = SIN(A)
141 X1 = E / (F + 2) * RADIUS + 100
144 Y2 = 1 / (F + 2) * RADIUS + 100
190 HPLOT X1,Y2 TO X2,Y4
270 A = M + A
350 GOTO 60

Key idea

A wireframe is a model made visible through projection. The program does not store a finished rectangle image. It computes points, draws edges between them, changes the angles, and computes again.
