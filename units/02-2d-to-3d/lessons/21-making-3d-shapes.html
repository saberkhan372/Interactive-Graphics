Unit 2 · Lesson 2.1

Making 3D Shapes

The question changes from where a point sits on the page to how an object occupies space. A cube has dimensions, orientation, and depth, and the code has to state those relationships clearly enough for the renderer to draw them.

Source deck: 2.1 - Making 3D shapes.pptx · 4 slides

What the class did

The warm-up asked how code can create 3D objects and what problems have to be solved. The deck itself is spare, but the curricular turn is clear: students are moving from drawing a picture to constructing an object.

In p5.js WebGL, primitives such as box, sphere, and plane give students quick access to 3D forms. The work is not merely calling those functions; it is understanding that translation, rotation, and scale now describe how the object sits in a coordinate system.

Code shown

function draw() {
  background(245);
  orbitControl();

  push();
  translate(-80, 0, 0);
  box(60, 120, 60);
  pop();
}

Key idea

A 3D primitive is not only a shape. It is a shape plus transformations. The object has to be placed, turned, and scaled before the scene can be read.

Try it
