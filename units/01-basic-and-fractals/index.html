Unit 1

BASIC and Fractals

The point of Applesoft BASIC is not nostalgia. The point is continuity: store a value, change it, test it, draw it again. That structure survives as the course moves from line numbers and GOTO into p5.js.

Students write real Applesoft BASIC on an Apple II emulator, with line numbers, GOTO, and HGR2, before ever touching p5.js. The point is not nostalgia: it is seeing that "draw a moving ball" is the same idea whether it is 1977 or now, just expressed differently. This unit also introduces recursion through a fractal tree, one of the hardest ideas in intro CS to make concrete.

Covers: BASIC on Apple II, more basics, a first project and presentation, the bouncing-ball bridge into p5.js, the fractal tree and remixing it, a class logo assignment, and the "Animating History with BASIC" project where students research and animate a real moment from graphics history.

Lesson 1.0

BASIC on Apple II

The unit begins on a 1977 personal computer and asks what graphics programming feels like when memory, speed, and display are small.
Lesson 1.1

More BASICs

Students move from the machine to the language: line numbers, loops, color, and the graphic commands that make Apple II drawing possible.
Lesson 1.2

First Project

The class logo assignment turns commands like GR, COLOR, PLOT, HLIN, and VLIN into an original graphic design problem.
Lesson 1.3

First Presentation

Students present the code before the image, then use the logo work to prepare for the bouncing-ball animation problem.
Lesson 1.4

Bouncing Ball

Motion becomes a loop: erase the old point, update the position, test the edges, draw the new point, and repeat.
Lesson 1.5

High Graphics Mode

Lo-res and hi-res Apple II modes make resolution and color tradeoffs concrete before the course turns to fractals.
Lesson 1.6

Remixing the Fractal Tree

Students change branches, generations, angles, randomness, and color to see recursion as a design space, not only a definition.
