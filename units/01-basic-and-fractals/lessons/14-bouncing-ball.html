Unit 1 · Lesson 1.4

Bouncing Ball

The bouncing ball is the course's bridge from static drawing to animation. It is small enough to understand, but it contains the full pattern: erase, update, test, draw, and repeat.

Source deck: 1.4 - Bouncing Ball.pptx · 8 slides

What the class did

The warm-up asked students to describe the main challenges in creating a bouncing-ball simulation in BASIC. The discussion named the necessary parts: position, velocity, old position, boundary checks, direction changes, and a loop that keeps the program alive.

The class then asked what was missing and what needed to be fixed. That framing matters: graphics programming is rarely a straight path from idea to image. The screen tells you what your model forgot.

Code shown

40 X = 20 : Y = 20
50 VX = RND(1) * 3 : VY = RND(1) * 3
80 PLOT PX, PY
100 PX = X : PY = Y
110 X = X + VX : Y = Y + VY
120 IF X > 39 OR X < 0 THEN GOSUB 1000
150 COLOR = 3 : PLOT X,Y
170 GOTO 70

Key idea

Motion comes from controlled forgetting. The program removes the old dot, remembers enough state to compute the next dot, checks whether the next dot is legal, and then draws again.
