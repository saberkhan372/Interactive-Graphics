Unit 1 · Lesson 1.6

Remixing the Fractal Tree

The fractal tree lesson makes recursion practical. Students do not only define recursion; they change the number of branches, the number of generations, the angle, the randomness, and the color until the repeated rule has a different character.

Source deck: 1.6 - Remixing the Fractal Tree.pptx · 7 slides

What the class did

The warm-up asked what a fractal tree is, how code creates one, and where the main challenges live. The lesson then organized the remix around questions: how to change branches or generations, how to accept user input as angles instead of radians, how to add randomness, how to use color, and how the same ideas might support other animations.

Dan Shiffman's Nature of Code appears here as a resource because the lesson is no longer only about old hardware. It is about how a simple rule becomes a natural-looking structure when it is repeated with variation.

Code shown

function branch(len) {
  line(0, 0, 0, -len);
  translate(0, -len);
  if (len > 4) {
    rotate(angle);
    branch(len * 0.67);
  }
}

Key idea

Recursion is a rule that calls itself with a smaller version of the problem. The tree looks complex because the rule repeats in space, not because the rule itself is complicated.
