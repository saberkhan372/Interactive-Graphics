Unit 1 · Lesson 1.3

First Presentation

The first presentation is a code-reading exercise as much as a design critique. Students show the code first because the image should be understood as the result of specific decisions, not only as a final screen.

Source deck: 1.3 - Presentation.pptx · 7 slides

What the class did

Students reflected on the logo they made, whether it matched what they hoped to make, and which challenges they had to solve. The presentation prompt asked them to show the code, point out interesting or difficult parts, name research or collaboration, show the graphic, and argue why it should become the class sticker.

The lesson then pivoted toward animation with BASIC. The class named the questions behind a bouncing ball: how to show motion, how to store position, how to change position, how to detect the edge, and how to reverse course.

Key idea

Animation is not a different kind of magic. It is the same drawing system with state added: position now has to persist, change, and be tested over time.
