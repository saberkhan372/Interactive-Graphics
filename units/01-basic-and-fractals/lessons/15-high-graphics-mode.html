Unit 1 · Lesson 1.5

High Graphics Mode

The jump from lo-res to hi-res is not simply an upgrade. More pixels and fewer, stranger color choices change what kind of image is reasonable to make and what kind of code the image demands.

Source deck: 1.5 - Fractal Tree.pptx · 7 slides

What the class did

The warm-up asked how original personal computers differ from modern computers and how older computers can still be used for graphics and animation. The lesson compared Apple II graphics modes: lo-res GR at 40 by 40 pixels with 16 nominal colors, and hi-res HGR at 280 by 160 pixels with 8 colors.

The new command was HPLOT, which made the higher-resolution screen programmable. This is the bridge toward a fractal tree: once the display can hold finer lines, repeated branching becomes visually legible.

Code shown

REM lo-res graphics
GR
PLOT 20, 20

REM hi-res graphics
HGR
HPLOT 140, 150 TO 140, 80

Key idea

Resolution is a creative constraint, not only a technical specification. A higher-resolution mode can make subtle structure possible, but it also changes the scale at which the program has to think.

Try it

Open Recursion Made Visible and set the reveal low. Each branch is a line, but the tree appears because the same line-drawing decision is repeated with changed scale and angle.
