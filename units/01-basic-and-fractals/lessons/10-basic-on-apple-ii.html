BASIC on Apple II

The first programming unit begins with a machine that makes its limits impossible to ignore. The Apple II is slow, small, and direct, which makes it useful for seeing how a program turns instructions into marks on a screen.

What the class did

Students reviewed the browser-file structure from Unit 0, then shifted to the Apple II: a 1977 personal computer, usually with 48 or 64 KB of RAM and a 1.022 MHz 6502 CPU. The lesson framed the machine as one of the first successful mass-produced microprocessor computers and then moved into emulator-based practice.

The class used the Apple IIjs emulator and an Applesoft BASIC reference. The point was to write code in the environment that shaped the graphics, not to imitate the era from a distance.

Key idea

Working inside an emulator keeps the historical constraint honest. Students can feel the scale of the machine in the commands, the display, the delay, and the amount of code required to make even a simple drawing.
