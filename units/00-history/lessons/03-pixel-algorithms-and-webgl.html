Unit 0 · Lesson 0.3

Pixel Algorithms and WebGL

The graph-paper image becomes an algorithmic problem in this lesson. Students have to decide what information a pixel needs, how a location should be named, and which instructions another person can actually follow.

Source deck: 0.3 - Pixel Algorithms and WebGL.pptx · 7 slides

What the class did

The warm-up asked students to explain what happens when the number of fish in a graphics example increases and why different computers perform differently. The class then returned to the pixel design: after drawing on graph paper, students wrote instructions for transmitting the design to someone else.

The coding turn introduced p5.js, the p5.js Web Editor, WebGL, and the Processing Foundation. The important bridge was not the syntax by itself. It was the move from human-readable directions on graph paper to executable directions in a browser.

Code shown

function setup() {
  createCanvas(400, 400, WEBGL);
}

function draw() {
  background(245);
  box(120);
}

Key idea

An algorithm has to be precise enough to survive translation. A person following directions and a browser running JavaScript are different executors, but both expose the same problem: unclear instructions produce unclear images.
