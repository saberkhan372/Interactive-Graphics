Unit 0 · Lesson 0.5

Processing Visit

The Processing Foundation material gives the course an important frame: p5.js is not only a library but also a public project about access to creative coding. The technical file structure and the social purpose of the tool belong in the same conversation.

Source deck: 0.5 - Processing Visit.pptx · 7 slides

What the class did

Students began by reflecting on the books and research materials they had examined and by preparing questions about the Processing Foundation. The lesson then returned to p5.js and WebGL coding, with explicit attention to the three files that make a browser sketch work.

The deck named HTML as structure, CSS as presentation, and JavaScript as behavior. That distinction matters because students are about to move between visual ideas, code files, and browser output repeatedly.

Code shown

<!-- index.html gives the page structure -->
<script src="https://cdn.jsdelivr.net/npm/p5/lib/p5.min.js"></script>
<script src="sketch.js"></script>

Key idea

A web sketch is not one object. It is a relationship among files: HTML gives the browser something to load, CSS can change how it is presented, and JavaScript can make the page draw, respond, and change over time.

Try it

Use the Pixel Algorithm Playground as a finished example of that structure. The controls are HTML, the visual system is CSS and canvas, and the rule that changes the grid is JavaScript.
