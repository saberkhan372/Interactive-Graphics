Welcome to Interactive Graphics

The course begins by treating computer graphics as a field with a definition, not as a collection of software tricks. A graphic has to be modeled before it can be rendered, and that distinction gives students a way to read every later tool.

Source deck: 0.0 - Welcome.pptx · 14 slides

What the class did

Students introduced themselves, defined computer graphics as the use of computers to synthesize and manipulate visual information, and considered why visual information carries so much cognitive weight. The deck then widened the frame: graphics depends on theory, including geometry, sampling, perception, and color, and on systems, including parallel processing and graphics-specific programming languages.

The first activity asked students to model and draw a 2D shape. That small task established the vocabulary for the course: what is the object, what information describes it, and how does the computer turn that description into an image?

Key idea

Before there is an image, there is a representation. The representation can be a list of points, a grid of pixels, a scene graph, or a rule. Rendering is the process of turning that representation into something visible.

Try it

Open the Constraint Machine and keep the subject fixed while the display system changes. The useful question is not which version looks best, but what information each system makes easy or difficult to preserve.
