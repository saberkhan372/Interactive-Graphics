Some History

This lesson treats computer graphics as part of a longer history of visual machines. Projection, persistence of vision, layered animation, parallax, chroma key, and pixels are not separate curiosities; each one is a way of making an image behave.

Source deck: 0.1 - Some History.pptx · 19 slides

What the class did

The warm-up compared two contemporary computers by price, CPU, GPU, operating system, and power use. Students were asked to connect those differences to graphics performance, making hardware visible before the history began.

The historical sequence moved from the Magic Lantern, Thaumatrope, Phénakistiscope, Praxinoscope, stop-motion, multiplane camera, and chroma key into early pixel displays. The lesson ended with a graph-paper pixel activity: create a design on a fixed grid, fill every pixel box, and prepare to describe the image as a system of instructions.

Key idea

Computer graphics inherits older visual problems. A screen is new hardware, but the central questions are older: how does an image move, how does a flat surface suggest depth, and how does a viewer understand a sequence of marks as one continuous visual event?

Try it

In the Constraint Machine, use the same subject across Apple II, NES, early web, and modern canvas modes. Write down one detail that disappears in each older system and one detail that becomes stronger because of the limit.
