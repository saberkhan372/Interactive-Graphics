Unit 0 · Lesson 0.7

Review HTML and JavaScript

The final lesson in the opening sequence turns reflection into setup. After following and improving pixel directions, students review the files that let a sketch run in the browser and type a small WebGL example themselves.

Source deck: 0.7 - Review HTML and JS.pptx · 8 slides

What the class did

The warm-up asked students how it felt to follow someone else's directions and whether those directions could be optimized. The lesson then reviewed HTML, CSS, and JavaScript, with special attention to the HTML file as the page the browser renders and the place where libraries and scripts are loaded.

The activity asked students to create a folder, make index.html and sketch.js, open the HTML file in a browser, and type out a WebGL box example rather than copying and pasting. That typing mattered: it made the file relationship, the function names, and the error surface visible.

Code shown

// sketch.js
function setup() {
  createCanvas(400, 400, WEBGL);
}

function draw() {
  background(220);
  rotateY(frameCount * 0.01);
  box(100);
}

Key idea

The browser project is a system of responsibilities. HTML loads the page and scripts, CSS can style the surface, and JavaScript supplies the behavior. Once that structure is visible, p5.js and WebGL become tools students can inspect rather than magic they have to trust.
