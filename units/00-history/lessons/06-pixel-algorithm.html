Unit 0 · Lesson 0.6

Pixel Algorithm

This lesson makes precision social. Students do not only write directions; they hand those directions to another person and find out whether the algorithm survives without the original author standing beside it.

Source deck: 0.6 - Pixel Algorithm.pptx · 8 slides

What the class did

Students brought printed steps for recreating their pixel drawings, exchanged directions with a partner, and tried to reproduce the design without seeing the original. They also recorded the thoughts and frustrations that came from following someone else's instructions.

The partner activity made ambiguity visible. Missing coordinates, unclear color names, inconsistent row order, or assumptions about the reader all became concrete because they changed the final image.

Code shown

for (let y = 0; y < rows; y++) {
  for (let x = 0; x < cols; x++) {
    let value = (x + y) % palette.length;
    fill(palette[value]);
    square(x * cell, y * cell, cell);
  }
}

Key idea

An algorithm is not clear because the author understands it. It is clear when another executor, human or machine, can follow it and produce the intended result.
