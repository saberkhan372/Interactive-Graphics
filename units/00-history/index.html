Unit 0

History and Constraint

The unit's central mechanism is constraint. Before the class calls an older system primitive or nostalgic, it asks what the machine could actually display, what it could not display, and what artists learned to do inside those limits.

Every era of computer graphics was defined by what its hardware would and would not let you do. Apple II gave you 1-bit black and white. The NES gave you a fixed, tiny palette. Bryce3D in the late 1990s gave a home computer real landscapes and pre-built models for the first time. This unit starts with that history, including the teacher's own path through it, from a childhood without computers to an NES from an aunt to a first PC, and ends with students testing the constraints directly rather than only reading about them.

Covers: some history, the history of graphics broadly, pixel algorithms, a personal pixel-algorithm project, a Processing Foundation guest visit, and an HTML/JS review.

The graph-paper activity becomes an algorithm problem: how to identify a pixel, describe it, and reproduce the image reliably.

Personal Pixel Algorithms

Students turn their own image into instructions and place technical questions beside research about artists, tools, and institutions.

Processing Visit

The Processing Foundation context connects p5.js to community, access, and the file structure of a browser sketch.
Lesson 0.6

Pixel Algorithm

Students exchange instructions and learn that an algorithm is only clear when someone else can execute it without the original image.

The unit closes by making the browser project structure explicit: HTML for structure, CSS for presentation, JavaScript for behavior.
