Unit 0 — History and Constraint

Pixel Algorithm Playground

One small rule, run once for every pixel. Change the rule, and the image changes because the system has different instructions for reading the grid.

The lesson is not that a pattern can be made quickly. The lesson is that a visual field can be described as coordinates, rules, and a palette, and that those three decisions are enough to make structure appear.

The rule runs for every cell and returns a number; the number picks a palette color. You can use:

x, ycell position, from top-left
cols, rowsgrid size
cx, cythe center cell
ddistance from center
ttime — turn on Animate

Try this

Start with the checkerboard. Change the 2 to a 3, then a 4. Predict the picture before you press enter.

Notice this

The rule never sees the whole picture. Every cell computes its own color from nothing but its position — and pattern still emerges.

Remix this

Keep your rule fixed and switch palettes, from 1-bit to Game Boy to the course palette. Which version of the same algorithm is easiest to read?
